# AI 知识库 APP 项目说明

## 产品目标
开发一个“对话式投喂型 AI 知识库 APP”。
第一版只做闭环：聊天投喂 + AI 自动整理 + 知识入库 + 知识问答 + 引用来源。

## 技术栈
- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Prisma
- PostgreSQL
- pgvector
- OpenAI API

## 代码规范
- 所有代码使用 TypeScript
- API 返回统一 JSON
- UI 先做简洁可用，不追求复杂动画
- 所有核心功能都要有 loading、empty、error 状态
- 不要一次性做高级功能，比如团队协作、知识图谱、语音、微信导入

## MVP 功能
1. 用户可以在聊天框中投喂知识
2. AI 自动生成标题、摘要、标签、分类、重要程度
3. 用户可以确认入库
4. 知识库列表可以查看、搜索、筛选
5. 用户可以基于知识库提问
6. 回答必须带引用来源

## 项目执行规范

本项目是 AI 知识库问答系统。Codex 执行任务时必须遵守以下规则。

### 基本原则

1. 不要重构，除非用户明确要求。
2. 不要修改与本次任务无关的页面。
3. 不要误改侧边栏、顶部搜索框、底部输入框、用户问题气泡、历史问答区域。
4. 不要删除已有功能。
5. 不要创建未被使用的新组件。
6. 修改前先定位真实生效的组件。
7. 修改后必须检查 git diff。
8. 不要自动 git commit。
9. 不要自动 git push。
10. 完成后只告诉用户是否可以提交。

### AI 问答 UI 保护规则

以下模块已经完成，除非用户明确要求，否则不要破坏：

1. AI 原始回答 Markdown 渲染。
2. CodeBlock 浅灰色代码块。
3. 代码块语言标签。
4. 代码块复制按钮。
5. 浅绿色「可直接复制给客户」模块。
6. 「复制客户答案」按钮。
7. AI 回答底部「复制 / 有帮助 / 没帮助」。
8. /chat?debugMarkdown=1 调试入口。

### Review 与 Verifiers 要求

完成每次修改后，必须进行代码自查和验证。

#### 一、diff 自查

请执行或检查：

1. git status
2. git diff --stat
3. git diff

自查要求：

1. 确认只修改了本次任务相关文件。
2. 确认没有误改无关 UI。
3. 确认没有删除已有功能。
4. 确认没有引入重复组件。
5. 确认没有把测试代码错误带入生产逻辑。
6. 如果发现无关修改，必须还原无关修改。

#### 二、运行 verifiers

请按项目实际情况运行：

1. npm run lint
2. npm run typecheck
3. npm run build

如果某个命令不存在，请说明“不存在，已跳过”，不要因此乱改 package.json。

如果本次涉及测试文件或已有测试命令，请运行相关测试。

#### 三、专项 review

每次修改后请检查：

1. 是否满足本次需求。
2. 是否改动范围过大。
3. 是否破坏现有功能。
4. 是否有 TypeScript 类型风险。
5. 是否有移动端溢出风险。
6. 是否有复制按钮失效风险。
7. 是否有 customerAnswer / finalAnswer 兼容问题。
8. 是否有 debugMarkdown=1 泄露到正式页面的问题。
9. 是否有性能问题。
10. 是否有未处理的异常分支。

#### 四、最终回复格式

完成后请按下面格式回复：

1. 修改文件：
- 文件 A：改了什么
- 文件 B：改了什么

2. Verifiers 结果：
- npm run lint：通过 / 失败 / 不存在
- npm run typecheck：通过 / 失败 / 不存在
- npm run build：通过 / 失败 / 不存在
- 其他测试：通过 / 失败 / 不存在

3. Review 结果：
- 是否发现无关修改：是 / 否
- 是否发现潜在问题：是 / 否
- 如有问题，说明是什么

4. 是否可以提交：
- 可以 / 不建议
- 原因
